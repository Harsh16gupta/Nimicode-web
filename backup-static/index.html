The Event Loop Lifecycle

Nimicode operates on an explicit, linear feedback loop. It runs local analysis, suggests concrete actions, triggers safety checks, and summaries results.

01

LLM Planning (Think)

Accepts your CLI context and runs a precise semantic planning session. Emits structured outputs outlining search targets or needed edits.

02

Tool Execution (Act)

Runs built-in tools (reading, writing, compiling, testing) to interact directly with your codebase. Every file operation generates standard diff outputs.

03

State Feedback (Respond)

Feeds tool outcomes (success logs, test errors, git diffs) back to the engine. Repeats loop until the goal resolves, providing a clean execution summary.

Built-in Capabilities

The Agent's Hands

Nimicode interacts with your files and compiler using robust, sandboxed command structures. Fully extensible via a TypeScript pluggable tool pattern.

read_file

Reads text content from workspace paths. Implements size limits and binary detection for safety.

Call Parameters
{ "path": "src/index.ts", "encoding": "utf-8" }

write_file

Atomically writes code blocks, creating folders implicitly. Forces validation checks to block syntax errors.

Call Parameters
{ "path": "test/unit.ts", "content": "describe('init', ...)" }

execute_command

Runs compiler tasks, linters, or test suites directly in shell, capturing exit codes and console errors.

Call Parameters
{ "cmd": "bun test", "timeoutMs": 30000 }

Local-First Security

Your Code Stays Yours.

Nimicode doesn't tunnel your prompts through secondary SaaS proxies. Your API key and code logic exist only on your local workstation.

Local API Storage

Credentials remain safely isolated in user home configuration config directory.

Direct Connections

CLI initiates HTTPS calls directly to the provider endpoints without middleman servers.
